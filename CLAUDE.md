# CLAUDE.md

This file provides guidance to Claude Code when working in the `cb_zinets` repository.

## What this repo is

`cb_zinets` is the ConceptBook domain app for Chinese characters. It generates a concept graph
from the ZiNets SQLite database and serves interactive concept books through the same Vite +
Vanilla JS + FastAPI stack as `~/projects/digital-duck/concept-book`.

## Commands

```bash
# Generate graph.yaml from the ZiNets database (all active characters)
python scripts/zinets_to_graph.py

# Generate with HSK subset (recommended for first runs)
python scripts/zinets_to_graph.py --set-id 300,100,30,10

# Generate graph.html navigator from graph.yaml
python scripts/concept_graph.py \
  --domain public/domains/chinese_characters/input/graph.yaml \
  visualize --format html \
  --output public/domains/chinese_characters/output/graph.html

# Validate graph (requires concept-book-press)
cd ~/projects/digital-duck/concept-book-press
python -B -m pipeline.cli validate \
  -i ~/projects/digital-duck/cb_zinets/public/domains/chinese_characters/input/graph.yaml

# Frontend dev (read-only, no book generation)
npm run dev          # http://localhost:5173/cb-zinets/

# Full stack (book generation requires spl123 conda env)
conda activate spl123
pip install -r requirements-api.txt
uvicorn api.app:app --reload --port 8000   # separate terminal
npm run dev
```

## Architecture

Mirrors `~/projects/digital-duck/concept-book`. Key differences:

- `base: '/cb-zinets/'` in `vite.config.js`
- Single domain: `public/domains/chinese_characters/`
- Extra script: `scripts/zinets_to_graph.py` (DB → graph.yaml)
- DB source: `~/projects/Proj-ZiNets/zinets_vis/dev_pg/backend/zinets_cache.sqlite`

## Domain directory layout

```
public/domains/chinese_characters/
  input/graph.yaml                      # generated by zinets_to_graph.py
  output/graph.html                     # generated by concept_graph.py
  output/{level}.{lang}/html/           # generated concept books
    book_{target}.html
    concept_{zi}.html
```

## HSK set_id values

| set_id | approx. count | meaning               |
|--------|---------------|-----------------------|
| '300'  | 250           | top-300 characters    |
| '100'  | 93            | top-100 characters    |
| '30'   | 31            | top-30 characters     |
| '10'   | 9             | top-10 characters     |
| ''     | ~10,908       | remaining (unclassified) |

## Applications tier

The `applications:` section in graph.yaml is reserved for 2-character and 4-character
idiomatic phrases. These require a separate data source (not yet in zn_zi) and will be
added in a future step.

## zinets_vis integration (Step 5)

The Vue.js zinets_vis app (`~/projects/Proj-ZiNets/zinets_vis`) will eventually surface
concept books when a user taps a character node. Design is deferred until the graph and
concept books are validated.
